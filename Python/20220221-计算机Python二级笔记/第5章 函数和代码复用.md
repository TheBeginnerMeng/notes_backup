## 5.4 代码复用

- 紧耦合：尽可能合理划分功能块，功能块内部耦合紧密
- 松耦合：模块间关系尽可能简单，功能块之间耦合度低。松耦合一般基于消息或协议实现，系统间交互简单
- 耦合性指的是程序结构中各个模块间相互关联的程度，取决于各个模块间接口的复杂程度和调用方式
